# 激光与振镜实施计划

## 1. 目标

在不修改现有云台控制逻辑的前提下，在 `demo` 中实现一条独立的激光 + 振镜控制链路。

## 2. 已确认决策

1. 协议版本：V1 文本帧。
2. 输入格式：`<board_id,x,y;...>`。
3. 执行模式：点到点定位 + 可选激光输出。
4. 坐标语义：V1 使用 REL（相对偏移）。
5. 安全互锁：仅当以下条件全部满足时允许出光：
   - 坐标有效
   - WARN 状态正常
   - ESTOP 已释放
6. 激光子系统与云台子系统保持独立。

## 3. 范围

包含：

1. USB 接收链路与激光坐标帧解析。
2. 振镜输出调度链路（复用 TIM7 + DMA1_Stream4）。
3. 激光状态机与互锁逻辑。
4. 激光子系统的 FreeRTOS 任务接入。

不包含：

1. 云台控制算法改动。
2. EZ3 DLL 内部机制逆向。
3. V1 阶段切换为二进制协议。

## 4. 架构设计

`application/laser_ctrl` 下模块划分：

1. `laser_proto.*`:
   - 解析 `<board_id,x,y;...>`
   - 做字段范围校验
   - 将合法点写入队列
2. `galvanometer.*`:
   - 将逻辑坐标映射为输出数据
   - 组织波形缓冲
   - 触发 TIM7 + DMA 输出
3. `laser.*`:
   - 状态机（IDLE、ARMED、EMIT、FAULT）
   - 执行安全互锁
4. `laser_task.*`:
   - 消费坐标队列
   - 调度振镜输出与激光门控
   - 采集运行统计

## 5. 接口约定（V1）

帧语法：

`<board_id,x,y;board_id,x,y;...>`

字段约束：

1. `board_id`：整数，范围 [0, 3]
2. `x`：浮点数，单位 mm，范围 [-1000.0, 1000.0]
3. `y`：浮点数，单位 mm，范围 [-1000.0, 1000.0]

错误处理：

1. 非法帧：丢弃并计数
2. 越界点：丢弃该点并计数
3. 解析缓冲溢出：重置解析状态并计数
4. 队列满：丢弃最旧点，保留最新点

## 6. 集成点

1. `USB_DEVICE/App/usbd_cdc_if.c`
   - 保持单一接收回调入口
   - 将负载分发给现有视觉解析与新增激光解析
2. `application/robot.c`
   - 增加激光子系统初始化
3. `application/robot_task.h`
   - 增加激光任务声明与创建
4. `Core/Src/stm32h7xx_it.c`
   - 在 USER CODE 区增加轻量 DMA 完成通知路径

## 7. 实施阶段

### 阶段 A - 骨架搭建

1. 创建 `application/laser_ctrl` 相关文件
2. 补充头文件与空实现
3. 在 `Makefile` 中加入新文件

退出标准：

1. 全量编译通过

### 阶段 B - 解析器 + 队列

1. 实现文本帧解析器
2. 实现坐标点队列
3. 增加解析统计计数

退出标准：

1. 合法帧可正确解析
2. 非法帧可丢弃且计数正确更新

### 阶段 C - 状态机

1. 实现 IDLE/ARMED/EMIT/FAULT 状态迁移
2. 实现三条件互锁
3. 互锁条件失效时强制立即关断

退出标准：

1. 故障注入后状态进入 FAULT 且输出被禁用

### 阶段 D - 驱动链路

1. 将坐标消费连接到振镜输出调度
2. 增加 DMA 完成同步机制

退出标准：

1. 在持续坐标流下输出序列稳定

### 阶段 E - 系统验证

1. 编译与冒烟测试
2. 协议健壮性测试
3. 长时间稳定性测试
4. 云台任务时序无回归

退出标准：

1. 全部验证项通过

## 8. 验证清单

1. `make -B -j4` 编译成功。
2. 合法/非法协议行为符合规范。
3. 互锁逻辑始终能快速关断激光输出。
4. 10 到 30 分钟连续数据流运行稳定。
5. 现有云台流程无可观测回归。

## 9. 风险与缓解

1. USB 回调冲突：
   - 缓解：保持单一回调入口并在内部做分发
2. ISR 负载过高：
   - 缓解：ISR 仅发送通知，不执行业务重逻辑
3. 解析洪峰：
   - 缓解：有界缓冲 + 队列覆盖策略
4. 坐标语义不匹配：
   - 缓解：V1 固定 REL，并预留模式扩展

## 10. 下一步

从阶段 A 开始，先接入所有新模块文件并确保工程可编译通过。
